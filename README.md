# CAPEC Automatic Labelling – SR-BH 2020

This repository contains the scripts and reproducibility materials developed for the Master’s Thesis:

**“Evaluación comparativa del etiquetado automático de patrones de ciberataque según CAPEC”**

The project evaluates several automatic and semi-automatic labelling techniques for web attack patterns using the **SR-BH 2020 multi-label dataset** and the **CAPEC taxonomy** as reference.

The goal of this work is not to create a new dataset or train a final supervised classifier. The goal is to compare different labelling approaches against the original CAPEC-based labels already provided in SR-BH 2020, using multilabel evaluation metrics.

## Repository purpose

This repository is provided to support the reproducibility of the experimental work described in the thesis.

It includes:

* Python scripts used during the experiment.
* Generated subsets derived from SR-BH 2020, when their size allows it.
* Predictions generated by the evaluated techniques.
* Evaluation tables and metrics.
* Figures used in the thesis.
* Complementary outputs for qualitative analysis.

## Techniques evaluated

The experiment compares five families of labelling techniques:

1. **Rules and heuristics**
   Explicit rules based on HTTP request patterns and CAPEC-related indicators.

2. **Similarity and embeddings**
   Sentence-transformer embeddings and cosine similarity between HTTP requests and CAPEC label prototypes.

3. **Weak supervision / distant supervision**
   Aggregation of weak labelling sources, including rules and embeddings.

4. **Zero-shot and few-shot LLM labelling**
   Local LLM-based labelling using a closed CAPEC label set, with zero-shot and few-shot prompts.

5. **Human-in-the-loop simulation**
   Prioritisation of instances for human review, with simulated correction using the original dataset labels as an oracle.

## Experimental workflow

The scripts are numbered according to the execution order used in the thesis.

### 1. Exploratory data analysis

```bash
python scripts/01_eda_srbh2020.py data_capec_multilabel.csv
```

This script generates initial information about label distribution, feature completeness and dataset structure.

Main outputs:

```text
results/eda_srbh2020/
```

### 2. Sampling and experimental partitions

```bash
python scripts/02_sampling_srbh2020.py data_capec_multilabel.csv
```

This script creates the experimental subsets used in the evaluation:

* `eval_set.csv`
* `train_set.csv`
* `llm_eval_subset_srbh2020.csv`

Main outputs:

```text
results/sampling_srbh2020/
```

### 3. Rules and heuristics

```bash
python scripts/03_rules_labeling_srbh2020.py results/sampling_srbh2020/eval_set.csv
```

Main outputs:

```text
results/rules_labeling_srbh2020/
```

### 4. Embeddings and similarity

```bash
python scripts/04_embeddings_labeling_srbh2020.py results/sampling_srbh2020/eval_set.csv --calibration-file results/sampling_srbh2020/train_set.csv
```

Main outputs:

```text
results/embeddings_labeling_srbh2020/
```

### 5. Weak supervision

```bash
python scripts/05_weak_supervision_srbh2020.py results/sampling_srbh2020/eval_set.csv --method weighted_vote
```

Main outputs:

```text
results/weak_supervision_srbh2020/
```

### 6. LLM zero-shot and few-shot labelling

Zero-shot:

```bash
python scripts/06_llm_labeling_srbh2020.py results/sampling_srbh2020/llm_eval_subset_srbh2020.csv --model qwen2.5:7b-instruct --prompt-mode zero_shot
```

Few-shot:

```bash
python scripts/06_llm_labeling_srbh2020.py results/sampling_srbh2020/llm_eval_subset_srbh2020.csv --model qwen2.5:7b-instruct --prompt-mode few_shot --examples-file results/sampling_srbh2020/train_set.csv
```

Main outputs:

```text
results/llm_labeling_srbh2020/
```

Note: this step requires a local Ollama installation and the selected model to be available locally.

### 7. Human-in-the-loop simulation

```bash
python scripts/07_human_in_the_loop_srbh2020.py results/sampling_srbh2020/eval_set.csv --simulate-oracle
```

Main outputs:

```text
results/human_in_the_loop_srbh2020/
```

This script does not implement a full annotation platform. It prioritises instances for review and simulates human correction using the original labels as an oracle.

### 8. Common evaluation

```bash
python scripts/08_evaluation_srbh2020.py results/sampling_srbh2020/eval_set.csv
```

Main outputs:

```text
results/evaluation_srbh2020/
```

This step computes common multilabel metrics for the evaluated techniques.

### 9. Final plots

```bash
python scripts/09_plots_results_srbh2020.py --input-dir results/evaluation_srbh2020 --out-dir results/plots_results_srbh2020
```

Main outputs:

```text
results/plots_results_srbh2020/
```

### 10. CAPEC/Normal inter-label matrices

```bash
python scripts/10_capec_confusion_matrices_srbh2020.py
```

Main outputs:

```text
results/plots_results_srbh2020/
```

## Evaluation metrics

The evaluation uses multilabel metrics, including:

* Micro precision, recall and F1.
* Macro precision, recall and F1.
* Hamming loss.
* Jaccard score.
* Subset accuracy.
* Cohen’s kappa.
* Per-label precision, recall, F1 and kappa.
* Agreement between methods.
* CAPEC/Normal inter-label matrices.

## Main result summary

In the evaluated scenario, the rule-based technique obtained the strongest overall performance on `eval_set`. This result is consistent with the presence of explicit technical patterns in many HTTP requests, such as SQL payloads, sensitive paths, abnormal HTTP methods and header manipulation indicators.

Embeddings showed weaker performance as an independent labelling technique. Weak supervision improved over embeddings but did not outperform rules. The human-in-the-loop strategy produced progressive improvements as the review budget increased. The few-shot LLM configuration improved over zero-shot, although LLM results were evaluated on a reduced subset due to computational cost.

These results are specific to SR-BH 2020 and the 12 CAPEC categories analysed in the thesis.

## Limitations

This repository and the associated experiment have several limitations:

* The evaluation is based on one dataset: SR-BH 2020.
* The original SR-BH 2020 labels are used as the experimental gold standard.
* The CAPEC scope is limited to the 12 categories present in the dataset.
* The LLM evaluation is performed on a reduced subset of 100 instances.
* The human-in-the-loop approach is simulated, not based on a real annotation campaign.
* No final supervised classifier is trained or deployed.

## Reproducibility notes

The experiment uses fixed seeds where applicable. The scripts are designed to generate outputs in a consistent format, especially prediction files using the `pred__<label>` naming convention.

Some paths may need to be adjusted depending on the local folder structure. The outputs included in this repository are provided to document the experimental results used in the thesis.

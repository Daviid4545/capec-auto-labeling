# CAPEC Automatic Labelling – SR-BH 2020

This repository contains the scripts and reproducibility materials developed for the Master’s Thesis:

**“Evaluación comparativa del etiquetado automático de patrones de ciberataque según CAPEC”**

The project evaluates several automatic and semi-automatic labelling techniques for web attack patterns using the **SR-BH 2020 multi-label dataset** and the **CAPEC taxonomy** as reference.

The goal of this work is not to create a new dataset or train a final supervised classifier. The goal is to compare different labelling approaches against the original CAPEC-based labels already provided in SR-BH 2020, using multilabel evaluation metrics.

## Repository purpose

This repository supports the reproducibility of the experimental work described in the thesis.

It includes:

* Python scripts used during the experiment.
* Generated experimental partitions derived from SR-BH 2020.
* Predictions generated by the evaluated techniques.
* Evaluation tables and metrics.
* Figures used in the thesis.
* Complementary outputs for qualitative analysis.

The `results/` directory contains the outputs generated during the execution of the experiment for the thesis. These outputs are included so that the results can be reviewed without rerunning the full workflow.

## Important execution note

The scripts are part of a sequential workflow and must be executed in order from the same working directory.

Each script creates its output folders using relative paths. Later scripts expect the outputs generated by previous scripts to be available in that same working directory. For example, the sampling script creates `sampling_srbh2020/`, and the following scripts use files from that folder. Similarly, weak supervision, human-in-the-loop, common evaluation and plotting scripts depend on prediction and metric files generated by earlier steps.

Therefore, changing the working directory between steps may cause later scripts to fail because the expected folders will not be found.

The Python scripts can be stored in the `scripts/` folder, but the commands must be launched from a consistent working directory. If the repository root is used as the working directory, commands should include the `scripts/` prefix. If the scripts are copied into the working directory, the usage shown inside each script can be used directly.

The `results/` directory included in this repository contains the outputs generated during the thesis experiment. It is provided for review and traceability, but the scripts do not automatically write inside `results/` unless that folder is used as the working directory.

## Techniques evaluated

The experiment compares five families of labelling techniques:

1. **Rules and heuristics**
   Explicit rules based on HTTP request patterns and CAPEC-related indicators.

2. **Similarity and embeddings**
   Sentence-transformer embeddings and cosine similarity between HTTP requests and CAPEC label prototypes.

3. **Weak supervision / distant supervision**
   Aggregation of weak labelling sources, including rules and embeddings.

4. **Zero-shot and few-shot LLM labelling**
   Local LLM-based labelling using a closed CAPEC label set, with zero-shot and few-shot prompts.

5. **Human-in-the-loop simulation**
   Prioritisation of instances for human review, with simulated correction using the original dataset labels as an oracle.

## Evaluation metrics

The evaluation uses multilabel metrics, including:

* Micro precision, recall and F1.
* Macro precision, recall and F1.
* Hamming loss.
* Jaccard score.
* Subset accuracy.
* Cohen’s kappa.
* Per-label precision, recall, F1 and kappa.
* Agreement between methods.
* CAPEC/Normal inter-label matrices.

## Main result summary

In the evaluated scenario, the rule-based technique obtained the strongest overall performance on the main evaluation set. This result is consistent with the presence of explicit technical patterns in many HTTP requests, such as SQL payloads, sensitive paths, abnormal HTTP methods and header manipulation indicators.

Embeddings showed weaker performance as an independent labelling technique. Weak supervision improved over embeddings but did not outperform rules. The human-in-the-loop strategy produced progressive improvements as the review budget increased. The few-shot LLM configuration improved over zero-shot, although LLM results were evaluated on a reduced subset due to computational cost.

These results are specific to SR-BH 2020 and the 12 CAPEC categories analysed in the thesis.

## Limitations

This repository and the associated experiment have several limitations:

* The evaluation is based on one dataset: SR-BH 2020.
* The original SR-BH 2020 labels are used as the experimental gold standard.
* The CAPEC scope is limited to the 12 categories present in the dataset.
* The LLM evaluation is performed on a reduced subset of 100 instances.
* The human-in-the-loop approach is simulated, not based on a real annotation campaign.
* No final supervised classifier is trained or deployed.

## Reproducibility notes

The experiment uses fixed seeds where applicable. The scripts are designed to generate outputs in a consistent format, especially prediction files using the `pred__<label>` naming convention.

The LLM step may produce slight differences depending on the local model version, Ollama version and execution environment, although the experiment uses a fixed prompt structure and low-temperature generation to improve reproducibility.
